# ⚡ MQTT 充电桩模拟 CLI 工具 — 使用说明

## 快速开始

### 1. 准备文件

把以下两个文件放在同一个文件夹：
- `mqtt_cli.exe` — 主程序
- `config.yaml` — 配置文件（可选，没有会自动生成）

### 2. 双击运行

双击 `mqtt_cli.exe`，看到以下菜单就说明启动成功：

```
⚡ MQTT 充电桩模拟 CLI v1.1.2
  环境: pre | 桩: XPAC2017YS03240002 | 速度: 2.0x

──────────────────────────────────────────────────
  主菜单
  [1] 单次跑充电订单
  [2] 批量跑充电订单
  [3] 场景脚本
  [4] 异常状态模拟
  [5] 单条报文发送
  [6] 预发充值钱包
  [7] 关闭订单
  [s] 设置（切换环境）
  [q] 退出
```

### 3. 按数字选择功能，按提示输入参数即可

## 功能说明

### [1] 单次跑充电订单

执行一次完整的充电流程。

操作步骤：
1. 选择充电模式：即插即充 / 扫码充电 / 即插即充（分步执行）
2. 输入桩编码（直接回车用默认值）
3. 输入 VIN 码（直接回车用默认值）
4. 扫码模式还需输入 UID

分步执行模式下，充电流程分为 7 个阶段，每步执行完暂停等待确认：
1. 上报空闲
2. 车辆验证
3. 启动状态（state 0→5）
4. BMS/YX/YcMeas 上报（可自定义 SOC、电压、电流等参数）
5. 结束充电
6. 交易上传
7. 恢复空闲

执行完成后会打印 `tradeID` 和 `orderID`，电池充检场景会用到。

### [2] 批量跑充电订单

循环执行多次充电流程，用于压力测试或批量造数。

额外输入：循环次数。执行过程中如果某轮失败，可以选择重试、跳过或停止。完成后会汇总成功/失败/跳过的轮数。

### [3] 场景脚本

#### 充电小结
模拟完整充电流程 + 多条不同功率 BMS 数据，验证充电小结卡片展示。

输入：桩编码、VIN、UID

#### 电池充检
模拟电池充检流程。

操作步骤：
1. 输入桩编码、VIN、UID
2. 工具自动跑充电到进行中（SOC<25）
3. 提示"请在 APP 执行充检"
4. 在手机 APP 上点击充检
5. 按回车，工具自动获取充检 ID（获取失败可手动输入）
6. 选择充检结果（检测完成/平台终止/BMS禁止充检/BEX1超时/暂停充电超时/脉冲输出超时/脉冲电流停止超时/充检时结束充电/其他错误）
7. 工具发送充检报文
8. 按回车结束充电

#### 充电需求功率满足度
模拟不同满足度场景，验证平台满足度展示。

三种模式：
- 需求低预期高（满足度 ≥ 95%）
- 车桩错配
- 同车分流/桩故障（满足度 < 95%）

发完一组 BMS 后可以选择继续发其他模式或结束充电。

#### 身份盗用
模拟车辆身份异常，验证平台告警和拦截。

四种模式：
- 正常（不触发告警）
- 电池类型不一致（触发告警+停用）— 可指定电池类型
- 蓄电池额定容量偏差 ratedAH（触发人工核实）— 可指定 ratedAH 值
- 蓄电池标定总能量偏差 ratedKWh（触发人工核实）— 可指定 ratedKWh 值

### [4] 异常状态模拟

模拟充电桩异常状态。

三种类型：
- 故障 — 发送故障遥信（errcode=E07）
- 急停 — 发送急停遥信（errcode=E05）
- 升级中 — 发送升级中遥信（status=6）

默认 5 秒发一次，发 3 次。发完可以选择继续发、切换异常类型或返回主菜单。

> 注：启动失败（state=255）、锁枪（E71）、离线等异常可通过命令行模式或 Kiro 自然语言调用。

### [5] 单条报文发送

单独发送某一类型的 MQTT 报文，支持 12 种类型：

| 序号 | 类型 | 说明 |
|------|------|------|
| 1 | yx | 遥信数据（状态、告警等） |
| 2 | ycBMS | BMS 数据（电压、电流、SOC） |
| 3 | ycMeas | 计量数据（电量） |
| 4 | ycAnalog | 采集数据 |
| 5 | starting | 启动状态 |
| 6 | chargEnd | 充电结束 |
| 7 | trade | 交易上传 |
| 8 | carChk | 车辆验证 |
| 9 | bootNoti | 启动通知 |
| 10 | pileProp | 桩属性 |
| 11 | cdProgress | 充检进度 |
| 12 | 自定义 JSON | 直接粘贴 JSON 发送 |

每种类型会提示输入对应参数，支持设置发送次数。

### [6] 预发充值钱包

给指定用户充值钱包余额（仅 pre 环境）。

输入：
- 用户 UID（默认使用配置文件中的 UID）
- 充值金额（单位：分，默认 9000000 = 9 万元）

### [7] 关闭订单

手动关闭一个充电订单。

输入：订单号（orderNo）

工具会自动根据当前环境选择对应的接口域名。

### [s] 设置

切换运行环境（pre / test），切换后会自动重连 MQTT 并更新默认桩编号。

## 快捷操作

| 输入 | 效果 |
|------|------|
| `q` | 返回主菜单 |
| `qq` | 直接退出程序 |
| 直接回车 | 使用默认值 |

执行完成后可以选择：
- `[1]` 再次执行同样的操作
- `[2]` 返回上一级菜单
- `[3]` 返回主菜单
- `[q]` 退出程序

## 命令行模式（非交互）

也可以通过命令行参数直接执行，不进入菜单：

```bash
# 即插即充
mqtt_cli.exe run plug --pile XPAC2017YS03240002 --vin TEST2K0Y5JI4P6BC7

# 扫码充电
mqtt_cli.exe run scan --pile XPAC2017YS03240002 --uid 8102985

# 批量跑 10 次
mqtt_cli.exe run plug --loop 10

# 充电小结
mqtt_cli.exe scenario summary --pile XPAC2017YS03240002 --uid 8102985

# 满足度（支持多模式逗号分隔）
mqtt_cli.exe scenario satisfaction --mode normal,mismatch,shunt

# 身份盗用
mqtt_cli.exe scenario identity-theft --mode bat-type

# 故障模拟
mqtt_cli.exe fault error --code E07

# 急停
mqtt_cli.exe fault estop

# 升级中
mqtt_cli.exe fault upgrading

# 使用 test 环境
mqtt_cli.exe --env test run plug
```

## 配置文件说明

`config.yaml` 可以修改默认参数，不用每次手动输入。

主要配置项：

```yaml
# 环境配置（可以新增自定义环境）
environments:
  pre:
    mqtt_ip: "47.96.240.241"     # MQTT 服务器地址
    mqtt_port: 12883              # MQTT 端口
    pile: "XPAC2017YS03240002"   # 默认桩编号
    vin: "TEST2K0Y5JI4P6BC7"     # 默认 VIN 码
    uid: "8102985"                # 默认用户 UID
  test:
    pile: "559847003"
    vin: "TESTNUYCXPKWVTIZF"
    uid: "1160057"

# 通用默认参数
defaults:
  env: "pre"        # 默认环境
  speed: 2.0        # 模拟速度倍数（越大越快）
  soc: 90           # BMS SOC
  bsoc: 20          # 开始 SOC
  esoc: 90          # 结束 SOC
  bat: 3            # 电池类型（3=磷酸铁锂 6=三元锂）

# 电池/BMS 参数（starting 报文用）
battery:
  maxAllowTemp: 105       # BMS 最高允许温度
  maxAllowVol: 427.6      # BMS 最高允许充电总电压
  ratedVol: 345.6         # 额定电压
  # ... 更多参数见 config.yaml
```

参数优先级：命令行参数 > 交互式输入 > 配置文件 > 内置默认值

修改配置文件后重新运行即可生效。

## 批量场景编排

将多个操作编排成 YAML 文件，一次性按顺序执行：

```bash
# 每日冒烟（即插即充 + 扫码充电）
python run_plan.py test_plans/daily_smoke.yaml

# 全量回归（所有场景 + 异常）
python run_plan.py test_plans/full_regression.yaml

# 只跑异常模拟
python run_plan.py test_plans/fault_tests.yaml

# 指定环境
python run_plan.py test_plans/daily_smoke.yaml --env test
```

执行完会输出汇总报告，显示每个步骤的成功/失败状态和总耗时。

自定义编排文件格式见 `test_plans/README.md`。

## 常见问题

**Q: 双击 exe 闪退怎么办？**
A: 在命令行（cmd）中运行 `mqtt_cli.exe`，可以看到具体错误信息。

**Q: MQTT 连接失败？**
A: 检查网络是否能访问 MQTT 服务器地址（默认 47.96.240.241:12883）。

**Q: tradeID 在哪里找？**
A: 每次充电流程完成后会醒目打印 `📋 tradeID: xxx | orderID: xxx`。日志文件也有记录，在 `logs/` 目录下。

**Q: 电池充检没有充检按钮？**
A: 确认以下条件：
1. 桩支持充检（工具会自动发 pileProp cdEn=1）
2. BMS 支持充检（工具会自动发 ycBMS cdFlag=2）
3. 车辆支持充检（数据库中 VIN 对应的 bat_type 等字段需要有值）
4. SOC < 25（工具默认设为 24）

**Q: 如何切换环境？**
A: 在主菜单按 `s` 切换，或修改 `config.yaml` 中的 `defaults.env`，或启动时用命令行参数 `mqtt_cli.exe --env test`。

**Q: 日志文件在哪？**
A: 在 exe 同目录下的 `logs/` 文件夹，按时间命名。
